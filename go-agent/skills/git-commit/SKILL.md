# Git 提交

## 目标

在提交前确认改动范围清晰、验证命令通过、敏感文件没有进入暂存区，并用简洁中文说明提交内容。默认提交信息使用中文，除非用户明确要求英文或项目已有强制英文规范。

## 使用流程

1. 查看状态：运行 `git status --short`，区分本次改动和无关改动。
2. 检查差异：运行 `git diff -- <path>` 或 `git diff --cached`，确认没有密钥、临时文件、无关重排。
3. 运行验证：优先执行项目已有测试、lint、build；没有统一命令时，执行和本次改动最相关的最小验证。
4. 暂存文件：只 `git add` 本次任务相关文件，不暂存 `.env`、密钥、日志、缓存和用户未要求的无关文件。
5. 提交信息：使用简洁中文标题。可以保留 Conventional Commit 前缀，例如 `feat: 添加 Git 提交技能`、`fix: 调整 DeepSeek 默认模型`。
6. 提交后确认：运行 `git status --short`，向用户报告 commit hash、验证结果和未提交的剩余改动。

## 注意事项

- 不使用 `git add .`，除非已经确认全部未暂存改动都属于本次任务。
- 不提交真实 API key；配置样例用 `.env.example`，真实配置留在被忽略的 `.env`。
- 不回滚用户已有改动；遇到不相关改动，只在报告中说明。
