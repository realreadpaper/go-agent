Task 14: 实现 capstone sfull 与 README

Prerequisites: Task 1-13 全部完成

Goal: 组装包含所有机制的完整 harness sfull，实现 REPL 命令，编写 README，运行 smoke tests。

给第一次接触 Agent 的读者

这一节是整合任务，不对应源教程中的单独一节。前面 13 个任务把机制拆开实现，是为了让每个概念都能被单独测试；现在要把它们组装成一个能演示完整能力的 sfull。

新手先懂这几个概念

  • 组装层：cmd/sfull 不应该重新实现业务逻辑，只负责创建 managers、注册工具、接线 hooks 和启动 REPL。
  • REPL 命令：除了自然语言输入，还需要 /tasks、/team、/inbox、/worktrees 这类直接观察状态的命令。
  • Smoke test：不验证每个细节，只验证完整程序能启动、基础命令能跑、核心 managers 能协同。
  • README 映射：读者需要知道每个 Go command 对应源中文教程哪一节，以及建议按什么顺序学习。

为什么这样收尾

教学项目最怕“每节都能跑，但读者不知道整体是什么”。Task 14 的价值是把 agent loop、工具、状态、团队和隔离目录串成一张总图，让第一次接触 agent 的读者看到完整 harness 如何从小机制组合出来。

完成后应该理解

你应该能从 README 反向讲清楚整条路线：最小 loop 如何长出工具、计划、记忆、持久任务、并发团队和隔离执行环境。

Files:

  • Create: go-agent/cmd/sfull/main.go
  • Create: go-agent/README.md
  • Test: go-agent/internal/integration/smoke_test.go

Context

本任务是最终整合，需要 Task 1-13 的所有 internal/* 包。

sfull 包含：

  • base tools (bash, read_file, write_file, edit_file)
  • todo
  • task subagent
  • load_skill
  • compact
  • task graph
  • background
  • team
  • protocols
  • autonomous
  • s12 worktree 工具 (默认启用，README 说明需要在 git 仓库内运行)

Steps

Step 1: 组装 sfull**

sfull 注册所有工具和 hook： 1. tools.Registry 注册 base tools + todo + task + load_skill + compact + task graph + background + team + protocols + autonomous + worktree。 2. agent.Loop 配置所有 BeforeCall 和 AfterTool hook。 3. REPL 支持用户输入和特殊命令。

Step 2: 实现 REPL 命令**

支持：

  • /compact
  • /tasks
  • /team
  • /inbox
  • /worktrees
  • /exit
Step 3: 编写 README**

内容：

  • 环境变量：AGENT_LLM_PROVIDER、AGENT_LLM_API_STYLE、AGENT_MODEL、AGENT_REASONING_EFFORT、OPENAI_API_KEY、OPENAI_BASE_URL、DEEPSEEK_API_KEY、DEEPSEEK_BASE_URL。
  • 每个 cmd 对应文档章节。
  • OpenAI GPT-5.5 快速开始：
cd go-agent
go test ./...
AGENT_LLM_PROVIDER=openai \
AGENT_LLM_API_STYLE=openai_responses \
AGENT_MODEL=gpt-5.5 \
OPENAI_API_KEY=... \
go run ./cmd/s01-agent-loop "List files"
  • DeepSeek V4 Pro 快速开始：
cd go-agent
AGENT_LLM_PROVIDER=deepseek \
AGENT_LLM_API_STYLE=anthropic_messages \
AGENT_MODEL=deepseek-v4-pro \
DEEPSEEK_API_KEY=... \
DEEPSEEK_BASE_URL=https://api.deepseek.com/anthropic \
go run ./cmd/s01-agent-loop "List files"
  • DeepSeek V4 Flash 快速开始：
cd go-agent
AGENT_LLM_PROVIDER=deepseek \
AGENT_LLM_API_STYLE=openai_chat \
AGENT_MODEL=deepseek-v4-flash \
DEEPSEEK_API_KEY=... \
DEEPSEEK_BASE_URL=https://api.deepseek.com \
go run ./cmd/s01-agent-loop "List files"
  • Provider 快速开始：OpenAI GPT-5.5、DeepSeek V4 Pro、DeepSeek V4 Flash。
  • API style 选择建议：OpenAI 默认 Responses；DeepSeek 默认 Anthropic-compatible。
  • Tool calling 映射说明：OpenAI function_call / Chat tool_calls / Anthropic tool_use 如何统一到内部 block。
  • 故障排查：401、404、unsupported model、tool schema rejected、reasoning items 未回传、DeepSeek unsupported field ignored。
  • 安全声明：OpenAI hosted tools 不启用，本项目只执行本地 harness tools。
Step 4: smoke tests**

Smoke tests 不调用真实 API，使用 fake client 覆盖：

  • 所有 cmd 包可编译。
  • s01 loop 可执行 fake 工具调用。
  • s07-s12 的文件状态机可在 temp dir 下运行。

Run:

cd go-agent
go test ./...
go test -race ./internal/background ./internal/team
go vet ./...

Expected: 全部 PASS。

git add go-agent/cmd/sfull go-agent/README.md go-agent/internal/integration
