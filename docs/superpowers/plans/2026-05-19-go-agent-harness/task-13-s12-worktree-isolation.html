Task 13: 实现 s12 worktree 任务隔离

Prerequisites: Task 8 (持久任务图)

Goal: 实现 git worktree 管理，任务与 worktree 双向绑定，支持创建、运行命令、保留和移除 worktree，操作事件写入 events.jsonl。

给第一次接触 Agent 的读者

源教程 s12 解决并行 agent 最实际的问题：多个任务不能都在同一个工作目录里改文件。任务图回答“做什么”，worktree 回答“在哪里做”。把两者绑定后，每个任务都能拥有自己的隔离目录和分支。

新手先懂这几个概念

  • Git worktree：同一个 Git 仓库可以签出多个工作目录，每个目录有自己的文件状态和分支。
  • 控制面：.tasks/ 记录任务状态、owner、绑定的 worktree 名称。
  • 执行面：.worktrees/ 下的隔离目录负责真实命令和代码修改。
  • 事件日志：events.jsonl 记录 create/remove/keep/failed，方便崩溃后理解发生过什么。

和源教程的对应关系

源 s12 把 task ID 和 worktree name 绑定。本任务用 internal/worktree/manager.go 封装 git worktree 命令，并在创建、执行、保留、移除时同步更新 task 状态和 worktree index。

完成后应该理解

你应该能说清为什么 worktree 是团队 agent 的安全阀：它让并行修改有边界，让完成、保留、删除都成为显式动作，而不是混在同一个脏工作区里。

  • Create: go-agent/cmd/s12-worktree-task-isolation/main.go

Context

本任务依赖 Task 8 的 tasks.Manager（任务与 worktree 绑定）。Worktree manager 调用 git worktree 命令实现隔离。

Steps

Step 1: 写 worktree manager 测试**

使用临时 git 仓库：

  • Create("auth-refactor", taskID) 调用 git worktree add -b wt/auth-refactor .worktrees/auth-refactor HEAD。
  • .worktrees/index.json 写入 worktree 记录。
  • task 写入 worktree，pending 变为 in_progress。
  • Remove(name, completeTask=true) 完成任务、解绑 worktree、写 events。

临时仓库创建：

git init
git config user.email test@example.com
git config user.name test
echo hello > README.md
git add README.md
git commit -m init
Step 2: 实现数据模型** 
type Worktree struct {
	Name   string `json:"name"`
	Path   string `json:"path"`
	Branch string `json:"branch"`
	TaskID int    `json:"task_id"`
	Status string `json:"status"`
}

.worktrees/index.json：

{"worktrees":[{"name":"auth-refactor","path":".worktrees/auth-refactor","branch":"wt/auth-refactor","task_id":1,"status":"active"}]}
Step 3: 实现事件流**

追加到 .worktrees/events.jsonl：

  • worktree.create.before
  • worktree.create.after
  • worktree.create.failed
  • worktree.remove.before
  • worktree.remove.after
  • worktree.remove.failed
  • worktree.keep
  • task.completed
Step 4: 注册 worktree 工具**

工具：

  • worktree_create(name, task_id)
  • worktree_run(name, command)
  • worktree_keep(name)
  • worktree_remove(name, force, complete_task)
  • worktree_list()
  • worktree_events(limit)

worktree_run 的 cwd 必须是 worktree path，不是主仓库。

Step 5: 实现 s12 CLI**

Run:

cd go-agent
go test ./internal/worktree ./internal/tasks
go run ./cmd/s12-worktree-task-isolation "Create worktree auth-refactor for task 1, then run git status in it."

Expected: 任务与 worktree 双向绑定；remove 可同时完成任务并移除 worktree。

Step 6: 实现后真实验证任务**

无网络验证要在临时 git 仓库里跑，确认不会污染当前工作区：

cd go-agent
go test -count=1 ./internal/worktree ./internal/tasks
go vet ./internal/worktree ./internal/tasks

真实 CLI 验证要在一个干净 git 仓库中创建 worktree、运行命令、保留或移除 worktree：

cd go-agent
go run ./cmd/s12-worktree-task-isolation "Create task 1 if needed, create worktree auth-refactor for it, run git status inside that worktree, then list worktree events."

Expected: .worktrees/index.json 和 .worktrees/events.jsonl 可观察；worktree_run 的 cwd 是隔离 worktree，不是主仓库。

Step 7: 提交** 
git add go-agent/internal/worktree go-agent/cmd/s12-worktree-task-isolation
git commit -m "feat(go): add worktree task isolation"
