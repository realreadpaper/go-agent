Task 7: 实现 s06 上下文压缩

Prerequisites: Task 2 (agent loop)

Goal: 实现三层上下文压缩：micro compact（截断旧工具结果）、auto compact（token 阈值触发 LLM 摘要）、manual compact（模型主动调用工具）。完整 transcript 落盘 JSONL。

给第一次接触 Agent 的读者

源教程 s06 解决上下文窗口有限的问题。Agent 工作越久，messages 里积累的工具输出越多。即使模型本身还能推理，太长的上下文也会让成本升高、注意力分散，最终无法继续。

新手先懂这几个概念

  • Micro compact：每轮静默处理，把很久以前的大块工具结果替换成短占位符，保留“做过什么”的痕迹。
  • Auto compact：估算 token 超过阈值后，保存完整 transcript，再让模型总结当前任务状态。
  • Manual compact：模型或用户主动调用 compact 工具，在关键节点压缩上下文。
  • Transcript：完整历史写入磁盘。活跃上下文变短，不等于历史不可恢复。

和源教程的对应关系

源 s06 展示三层压缩策略。本任务用 internal/compact/manager.go 统一管理 token 估算、旧结果替换、摘要调用和 JSONL transcript。Loop 只通过 before hook 调用它，仍保持主流程稳定。

完成后应该理解

你应该能说清“压缩”不是简单删除，而是在保留可恢复历史的前提下，给模型一个更短、更有用的工作记忆。

Context

本任务依赖 Task 2 的 agent.Loop 和 BeforeCallHook 机制。compact manager 作为 hook 注入 loop，不修改 loop 主流程。

Steps

Step 1: 写 micro compact 测试**

构造 6 个 tool_result，KeepRecent=3。执行后前 3 个长结果变为 [Previous: used ]，最近 3 个保持原文。

Step 2: 实现 token 估算**

EstimateTokens(messages []llm.Message) int：将 JSON 长度除以 4，作为教学实现的近似值。

Step 3: 实现 auto compact**

Manager.AutoCompact(messages)：

  • 保存完整 JSONL 到 .transcripts/transcript_.jsonl。
  • 调用 llm.Client 生成摘要。
  • 返回单条 user message：[Compressed]\n\n。
Step 4: 注册 manual compact 工具**

因为工具 handler 需要修改 loop 外部状态，用 Manager.RequestManual() 设置标记；下一轮工具执行后由 hook 替换 messages。

Step 5: 实现 s06 CLI**

BeforeCall 顺序： 1. MicroCompact 2. AutoCompactIfNeeded

Run:

cd go-agent
go test ./internal/compact
go run ./cmd/s06-context-compact "Use the compact tool to manually compress the conversation"

Expected: 测试 PASS；.transcripts/ 写入 JSONL。

Step 6: 提交** 
git add go-agent/internal/compact go-agent/cmd/s06-context-compact
git commit -m "feat(go): add context compaction"
