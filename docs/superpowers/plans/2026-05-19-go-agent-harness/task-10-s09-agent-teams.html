Task 10: 实现 s09 团队邮箱与持久队友

Prerequisites: Task 2 (agent loop)

Goal: 实现基于 JSONL inbox 的消息总线和持久队友。每个队友运行独立 goroutine loop，通过邮箱异步通信。

给第一次接触 Agent 的读者

源教程 s09 从单个 agent 扩展到团队。Subagent 是一次性的，后台任务只能跑命令；团队里的 teammate 则是有名字、有角色、有生命周期的 agent loop，可以跨多轮收消息、回消息、继续工作。

新手先懂这几个概念

  • Teammate：一个独立运行的 agent loop，有自己的 messages、角色和状态。
  • Roster：.team/config.json 记录团队成员和状态，让主 agent 知道谁存在。
  • Inbox：每个成员一个 JSONL 文件，别人发送消息就是追加一行。
  • Drain-on-read：读取 inbox 后清空，避免同一条消息反复进入上下文。

和源教程的对应关系

源 s09 用 JSONL inbox 做文件级消息总线。本任务用 internal/team/bus.go 管消息追加和读取，用 internal/team/manager.go 管队友 goroutine、状态和生命周期。

完成后应该理解

你应该能把团队协作看成多个 agent loop 通过磁盘消息队列协调。它不是共享一个大上下文，而是每个 agent 保持自己的上下文，只通过显式消息交换信息。

Context

本任务依赖 Task 2 的 agent.Loop。每个队友是一个独立的 loop 实例，运行在自己的 goroutine 中。

Steps

Step 1: 写 MessageBus 测试**

覆盖：

  • Send append JSON line 到 .team/inbox/.jsonl。
  • ReadInbox(name) 返回全部消息并清空文件。
  • 空 inbox 返回 []。
Step 2: 实现 team config**

.team/config.json：

{"members":[{"name":"alice","role":"coder","status":"working"}]}

manager 内存中维护 goroutine cancel 函数和状态锁。

Step 3: 实现队友 loop**

每个队友：

  • 独立 messages。
  • 每次 LLM 调用前读自己的 inbox。
  • 结束时状态改为 idle。
Step 4: 注册团队工具**

工具：

  • spawn_teammate(name, role, prompt)
  • list_teammates()
  • send_message(to, content)
  • broadcast(content)
  • read_inbox(name)
Step 5: 实现 s09 CLI**

支持 REPL 命令：

  • /team 输出 config。
  • /inbox drain lead inbox。

Run:

cd go-agent
go test ./internal/team
go run ./cmd/s09-agent-teams "Spawn alice as coder and bob as tester. Have alice send bob a message."

Expected: .team/config.json 和 .team/inbox/*.jsonl 可观察。

Step 6: 提交** 
git add go-agent/internal/team go-agent/cmd/s09-agent-teams
git commit -m "feat(go): add persistent agent teams"
