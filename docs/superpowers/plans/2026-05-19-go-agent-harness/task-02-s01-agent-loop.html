Task 2: 实现 s01 agent loop 和 bash 工具

Prerequisites: Task 1 (LLM 抽象层)

Goal: 实现稳定的 agent loop、工具注册表和 bash 工具，组成最小可运行的 s01 命令。

给第一次接触 Agent 的读者

源教程 s01 的核心句子是：一个循环加一个工具，就已经是最小 agent。这里的“agent”不是神秘的新模型，而是一段程序：它把用户请求发给 LLM，看到模型要调用工具时替它执行，再把工具结果放回对话，直到模型不再请求工具。

新手先懂这几个概念

  • Agent loop：不断重复“调用模型 -> 执行工具 -> 写回结果”的控制流。loop 是 harness 的心脏，后续章节都尽量不改它。
  • stop_reason：模型告诉 harness 自己为什么停下。如果是 tool_use，说明要执行工具；否则说明可以把回答交给用户。
  • ToolRunner：Go 里的接口，屏蔽具体工具实现。loop 只知道“列出工具”和“按名称运行工具”。
  • bash 工具：最粗粒度但最有用的工具，可以跑命令、看文件、执行测试，也是安全边界最需要谨慎的地方。

和源教程的对应关系

源 s01 用 Python 展示 while True 循环和单个 bash 工具。本任务把同样的机制落到 Go：internal/agent/loop.go 放循环，internal/tools/registry.go 放工具注册表，cmd/s01-agent-loop 提供可运行入口。

完成后应该理解

你应该能画出一次完整工具调用的路径：用户 message 进入模型，模型返回 tool_use，Go 执行 bash，tool_result 作为新的 user message 回到模型。只要这条路径稳定，后面所有复杂能力都有地方挂载。

Context

本任务依赖 Task 1 完成的 internal/llm 包。核心类型见 overview 核心数据模型：

Steps

Step 1: 写 loop 测试**

场景：

  • fake client 第一轮返回 tool_use，工具结果被追加为 user message。
  • fake client 第二轮返回文本，loop 退出。
  • MaxRounds 防止无限工具调用。

Run:

cd go-agent
go test ./internal/agent

Expected: 先 FAIL，提示 Loop 未定义。

Step 2: 实现 loop 与 registry**

按 overview 核心数据模型中的代码实现 Loop.Run 和 Registry。工具结果内容截断到 50,000 字符。

Step 3: 实现 bash 工具**

internal/tools/base.go：

  • RegisterBash(reg, workdir string)。
  • 使用 exec.CommandContext(ctx, "bash", "-lc", command)。
  • timeout 120 秒。
  • 拦截危险片段：rm -rf /、sudo、shutdown、reboot、> /dev/。
Step 4: 实现 s01 CLI**

cmd/s01-agent-loop/main.go：

  • 从命令行参数或 stdin 读取 prompt。
  • 调用 llm.DefaultConfigFromEnv() 和 llm.NewClient()，构造 provider-specific client、tools.Registry、agent.Loop。
  • 注册仅 bash。
  • system prompt 说明当前 workdir 和 bash 工具。

Run:

cd go-agent
go test ./...
go run ./cmd/s01-agent-loop "What is the current git branch?"

Expected: 测试 PASS；运行命令时如未配置 API key，打印清晰错误。

Step 5: 实现后真实验证任务**

先跑无网络回归，确认 loop、hook、bash 工具和 provider adapter 都能编译通过：

cd go-agent
go test -count=1 ./...
go vet ./...

再用真实模型验证 agent loop 能完成一次工具调用闭环。这个 prompt 必须迫使模型调用 bash，不能只靠模型常识回答：

cd go-agent
go run ./cmd/s01-agent-loop "What is the current git branch? Use bash."

Expected: 输出包含当前分支名；调试日志能看出至少一次 tool_use:bash 和一次 tool_result。如果 API key 缺失，程序应明确提示缺少哪个环境变量。

Step 6: 提交** 
git add go-agent/internal/agent go-agent/internal/tools go-agent/cmd/s01-agent-loop
git commit -m "feat(go): implement s01 agent loop"
