Task 12: 实现 s11 自主队友

Prerequisites: Task 10 (团队邮箱), Task 8 (持久任务图)

Goal: 实现自主队友的 IDLE phase：轮询 inbox 和任务板，自动认领 ready 任务，超时无任务则 shutdown。支持身份重注入保持上下文。

给第一次接触 Agent 的读者

源教程 s11 让队友从“被分配任务”变成“自己找活干”。到这一节，任务板已经持久化，团队也能通信。自治的关键是空闲阶段：队友做完当前工作后，不立刻退出，而是看看 inbox 和任务板有没有新工作。

新手先懂这几个概念

  • WORK phase：队友正在运行普通 agent loop，调用工具完成当前任务。
  • IDLE phase：模型停止工具调用后，harness 暂时接管，周期性检查 inbox 和 ready tasks。
  • 自动认领：找到 pending、无 owner、无阻塞的任务后，把 owner 写成自己并恢复 WORK。
  • 身份重注入：上下文压缩后，队友可能丢失“我是谁”的线索，harness 需要补回身份块。

和源教程的对应关系

源 s11 展示 WORK/IDLE 生命周期。本任务把空闲逻辑放进 internal/autonomous/idle.go，并和 tasks.Manager、team.Bus 协作，确保自动认领可以被单元测试覆盖。

完成后应该理解

你应该能解释自治不是让模型无限自由发挥，而是给它一个受控循环：没活就等，有消息就处理，有 ready task 就认领，超时就退出。

Context

本任务同时依赖 Task 10 的 team.Manager（队友 goroutine）和 Task 8 的 tasks.Manager（任务板）。自主认领需要两者协同。

Steps

Step 1: 写任务认领测试**

覆盖：

  • 只认领 pending、owner == ""、blockedBy == [] 的任务。
  • 认领后 task status 变为 in_progress，owner 写入队友名。
  • 已阻塞任务不被认领。
Step 2: 实现 IDLE phase** 
type IdleConfig struct {
	PollInterval time.Duration
	Timeout      time.Duration
}

流程： 1. 每 5 秒读 inbox。 2. inbox 非空，注入 ... 并回 WORK。 3. 扫描 ready unclaimed task。 4. 找到后 claim，并注入 Task #N: subject。 5. 60 秒无事可做则 shutdown。

Step 3: 实现 idle 和 claim_task 工具**

idle 工具让模型主动进入空闲阶段。claim_task(task_id) 手动认领任务，复用 TaskManager 校验。

Step 4: 实现身份重注入**

当队友 messages 长度小于等于 3 时，在开头插入：

<identity>You are 'alice', role: coder, team: default. Continue your work.</identity>
Step 5: 实现 s11 CLI**

支持 /tasks 和 /team。

Run:

cd go-agent
go test ./internal/autonomous ./internal/tasks ./internal/team
go run ./cmd/s11-autonomous-agents "Create 3 tasks on the board, then spawn alice and bob. Watch them auto-claim."

Expected: 队友能从 .tasks/ 自动认领任务。
