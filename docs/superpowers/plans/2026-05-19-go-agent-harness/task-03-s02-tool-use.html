Task 3: 实现 s02 工具分发、文件工具和路径沙箱

Prerequisites: Task 2 (agent loop + registry)

Goal: 在工具注册表中添加 read_file、write_file、edit_file，全部路径限制在 workdir 沙箱内。

给第一次接触 Agent 的读者

源教程 s02 的重点是：加工具不应该改 agent loop。s01 只有 bash，模型能做很多事，但所有事情都绕到 shell，既难控制又难测试。s02 把常见文件操作变成专用工具，让 harness 可以在工具层加路径沙箱、参数校验和稳定输出。

新手先懂这几个概念

  • 工具分发：模型输出工具名和参数，harness 用工具名在 registry 中找到 handler。这个查表过程代替一长串 if/else。
  • 专用工具：read_file 比 cat 更可控，write_file 比 shell 重定向更容易测试，edit_file 可以限制必须命中旧文本。
  • 路径沙箱：任何文件路径都先转换成 workdir 内的绝对路径，禁止 ../ 逃到项目外。
  • 稳定返回字符串：工具 handler 不直接写 stdout，而是返回字符串给 loop，再由 loop 组装为 tool_result。

和源教程的对应关系

源 s02 强调 dispatch map。本任务用 Go 的 Registry 表达同一个思想：新增工具就是注册一个 ToolSpec 和一个 Handler，不碰 Loop.Run 的主流程。

完成后应该理解

你应该能解释为什么工具越具体，agent 越可靠；为什么路径安全必须放在工具边界；以及为什么“loop 不变、工具可扩展”是后续 harness 演进的关键。

  • Modify: go-agent/internal/tools/base.go
  • Create: go-agent/cmd/s02-tool-use/main.go
  • Test: go-agent/internal/tools/base_test.go

Steps

Step 1: 写路径沙箱测试**

覆盖：

  • read_file 可读 workdir 内文件。
  • ../outside.txt 返回 path escapes workspace。
  • write_file 自动创建父目录。
  • edit_file 只替换第一次出现的 old_text。
Step 2: 实现 SafePath 与文件工具**

Go 实现点：

  • filepath.Abs + filepath.Rel 判断路径没有 .. 前缀。
  • read_file 支持 limit 行数。
  • 文件读写使用 os.ReadFile、os.WriteFile。
  • edit_file 找不到文本时返回 Error: text not found in 。
Step 3: 实现 s02 CLI**

注册 bash、read_file、write_file、edit_file。

Run:

cd go-agent
go test ./internal/tools
go run ./cmd/s02-tool-use "Create a file called greet.go with a greet function, then read it"

Expected: 测试 PASS；CLI 可让模型调用文件工具。

Step 4: 实现后真实验证任务**

无网络验证要覆盖路径沙箱和文件工具稳定输出：

cd go-agent
go test -count=1 ./internal/tools ./internal/agent
go vet ./internal/tools ./internal/agent

真实模型验证要让模型依次写文件、读文件、编辑文件，并确认 ../ 逃逸会被拒绝：

cd go-agent
go run ./cmd/s02-tool-use "Create tmp/greet.go with a greet function, read it, edit it to add a Chinese comment, then try to read ../outside.txt and report the sandbox error."

Expected: tmp/greet.go 存在且内容被编辑；输出中能看到 read_file、write_file、edit_file 的调用结果，以及路径逃逸错误。

git add go-agent/internal/tools go-agent/cmd/s02-tool-use
git commit -m "feat(go): add file tools and dispatch"
