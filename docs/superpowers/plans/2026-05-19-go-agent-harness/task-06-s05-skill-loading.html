Task 6: 实现 s05 Skill 加载

Prerequisites: Task 3 (文件工具和路径沙箱)

Goal: 实现按需 skill 加载，从磁盘扫描 skills/**/SKILL.md，解析 YAML frontmatter，通过 load_skill 工具注入完整 skill 内容；同时提供 create_skill 工具，把一次可复用经验自动沉淀为新的 SKILL.md。

给第一次接触 Agent 的读者

源教程 s05 讨论“知识什么时候给模型”。如果把所有工作流、代码审查清单、框架说明都塞进 system prompt，模型每一轮都要带着大量无关 token。Skill loading 的做法是先只告诉模型有哪些 skill，等模型真的需要时再加载完整内容。

新手先懂这几个概念

  • Skill 描述层：系统提示中只放名称和一句描述，成本低，帮助模型知道可选能力。
  • Skill 内容层：模型调用 load_skill 后，完整 SKILL.md 通过 tool_result 进入上下文。
  • Frontmatter：SKILL.md 顶部的 YAML 元数据，用来声明 name、description 等可索引信息。
  • 按需注入：不是 harness 主动猜，而是让模型根据任务显式请求。
  • 经验沉淀：当某个流程以后还会重复使用时，模型可以调用 create_skill 生成新的 skill，下次运行时会被扫描并出现在 skill 列表中。

和源教程的对应关系

源 s05 使用两层提示模型。本任务用 internal/skills/loader.go 扫描磁盘 skill，用 gopkg.in/yaml.v3 解析 frontmatter，再注册 load_skill 工具。为了让同一次开发中的经验能被后续复用，本任务还会增加 skill 生成和更新逻辑：create_skill 接收 name、description、content，校验名称只包含小写字母、数字和连字符，写入 skills/<name>/SKILL.md，并默认拒绝覆盖已有 skill；update_skill 只允许更新当前目录中已经存在的 skill，避免模型因为拼写相近而意外创建新目录。

完成后应该理解

你应该能解释为什么 skill 不是普通配置文件：它是可被模型按需加载的操作指南。好的 skill loading 让 agent 拥有更多领域能力，同时不让每轮上下文膨胀。你也应该能解释为什么自动生成 skill 要受限：harness 只允许写入 skills 目录、拒绝路径穿越、拒绝无 frontmatter 的草稿，避免模型把临时日志或敏感内容随意写成长期知识。

Step 1: 复制示例 skills**

从 /Users/hejianglong/Desktop/github/learn-claude-code/skills 复制到 go-agent/skills，保持目录名不变。

Step 2: 写 loader 测试**

覆盖：

  • 递归扫描 SKILL.md。
  • YAML frontmatter 中 name 优先于目录名。
  • Descriptions() 只输出名称和 description。
  • Load("missing") 返回 unknown skill 错误文本。
Step 3: 实现 loader**

使用 gopkg.in/yaml.v3 解析 frontmatter。Load(name) 返回：

<skill name="code-review">
...
</skill>
Step 4: 注册 load_skill 工具**

system prompt 包含：

Skills available:
  - agent-builder: ...
  - code-review: ...

完整 Skill 内容只通过 tool result 注入。

Run:

cd go-agent
go test ./internal/skills
go run ./cmd/s05-skill-loading "What skills are available?"

Expected: 测试 PASS；CLI 能列出可用 skill。

Step 5: 增加 create_skill 经验沉淀工具**

实现方案：

  • 新增 SkillDraft，包含 name、description、content、overwrite。
  • name 会规范化为小写连字符格式，例如 Git Commit Safety 变成 git-commit-safety；空名称或无法规范化的名称返回错误。
  • 写入路径固定为 skills/<name>/SKILL.md，不接受模型提供任意路径。
  • 默认拒绝覆盖已有 SKILL.md；只有 overwrite=true 时才允许替换。
  • 生成内容必须包含 YAML frontmatter：name 和 description，正文由模型给出的可复用步骤组成。
  • 写入成功后更新当前 loader 索引，让同一轮 agent 可以立刻 load_skill 新 skill。
  • 新增 update_skill 工具专门优化已有 skill：它要求目标 SKILL.md 已存在，写入完整替换内容，然后刷新 loader 索引。

工具 schema 示例：

{
  "name": "create_skill",
  "input": {
    "name": "git commit safety",
    "description": "Use when preparing git commits or checking staged changes",
    "content": "# Git Commit Safety\n\n## Overview\n...",
    "overwrite": false
  }
}

更新已有 skill 的工具 schema 与 create_skill 类似，但工具名是 update_skill，并且目标 skill 不存在时必须返回清晰错误：

{
  "name": "update_skill",
  "input": {
    "name": "git-commit",
    "description": "Use when preparing git commits or checking staged changes",
    "content": "# Git Commit\n\n## Overview\n...",
    "overwrite": false
  }
}
Step 6: 实现后真实验证任务**

无网络验证要覆盖 skill 发现、frontmatter 解析、触发规则和缺失文件错误：

cd go-agent
go test -count=1 ./internal/skills ./internal/tools
go vet ./internal/skills

真实模型验证要让模型先列 skill，再加载一个本地 skill，并在回答中说明它使用了哪条本地规则；再要求模型把本次经验生成一个新 skill，并立刻加载确认：

cd go-agent
go run ./cmd/s05-skill-loading "List available skills, load the git-commit skill if present, and summarize its commit safety rules in Chinese."
go run ./cmd/s05-skill-loading "Create a reusable skill named demo-repeatable-check for checking git status before commits, then load it and summarize its path."

Expected: 输出包含 skill 名称、路径和中文规则摘要；不存在的 skill 应返回清晰错误而不是 panic；生成 skill 后 skills/demo-repeatable-check/SKILL.md 存在，且同一轮可以被 load_skill 读取；更新已有 skill 时必须调用 update_skill，并在更新后再次 load_skill 验证内容。
