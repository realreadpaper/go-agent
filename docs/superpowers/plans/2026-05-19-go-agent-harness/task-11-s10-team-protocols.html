Task 11: 实现 s10 request-response 团队协议

Prerequisites: Task 10 (团队邮箱与持久队友)

Goal: 在团队通信基础上添加结构化 request-response 协议，支持 shutdown 和 plan approval 两种 kind，通过 FSM 跟踪请求状态。

给第一次接触 Agent 的读者

源教程 s10 解决“消息有了，但协商没有规矩”的问题。普通消息适合通知，却不适合需要确认的动作，比如请求队友关机、审批高风险计划。协议给这些动作加上 request ID 和状态机。

新手先懂这几个概念

  • Request ID：每个请求一个唯一 ID，响应必须引用同一个 ID，才能把来回两条消息配对。
  • FSM：有限状态机。这里最小状态是 pending -> approved 或 pending -> rejected。
  • Shutdown protocol：领导请求队友优雅退出，队友可以先收尾再批准。
  • Plan approval：队友提交计划，领导批准或驳回，避免模型直接执行高风险变更。

和源教程的对应关系

源 s10 把 shutdown 和 plan review 都建模成同一种 request-response。本任务用 internal/protocols/fsm.go 承载状态转换，让团队工具只负责发送消息和更新请求状态。

完成后应该理解

你应该能看出协议的价值不是多几个工具，而是让 agent 之间的协作可追踪、可测试、可恢复。只要有 request ID，后续就能审计“谁请求了什么、谁同意了什么”。

Context

本任务依赖 Task 10 的 team.Manager 和 team.MessageBus。协议请求通过 MessageBus 传递，FSM 跟踪状态转换。

Steps

Step 1: 写 FSM 测试**

覆盖：

  • 新请求状态为 pending。
  • approve 后状态为 approved。
  • reject 后状态为 rejected。
  • unknown request id 返回错误。
Step 2: 实现通用请求模型** 
type Request struct {
	ID      string         `json:"id"`
	Kind    string         `json:"kind"`
	From    string         `json:"from"`
	To      string         `json:"to"`
	Status  string         `json:"status"`
	Payload map[string]any `json:"payload"`
}

支持 kind:

  • shutdown
  • plan_approval
Step 3: 注册协议工具**

工具：

  • shutdown_request(teammate)
  • shutdown_response(request_id, approve, reason)
  • plan_submit(plan)
  • plan_review(request_id, approve, feedback)

所有请求/响应通过 MessageBus 传递，并在 tracker 中更新状态。

Step 4: 实现 graceful shutdown**

队友收到 approved shutdown 后：

  • 完成当前工具轮。
  • 发送响应。
  • 状态改为 shutdown。
  • goroutine 返回。
Step 5: 实现 s10 CLI**

Run:

cd go-agent
go test ./internal/protocols ./internal/team
go run ./cmd/s10-team-protocols "Spawn alice as coder, then request her shutdown."

Expected: shutdown request 有 request_id，状态从 pending 到 approved/rejected。

Step 6: 实现后真实验证任务**

无网络验证要覆盖 request id、pending/approved/rejected 状态机和 inbox 事件顺序：

cd go-agent
go test -count=1 ./internal/protocols ./internal/team
go vet ./internal/protocols ./internal/team

真实模型验证要发起一次 shutdown request，并让目标队友批准或拒绝：

cd go-agent
go run ./cmd/s10-team-protocols "Spawn alice, request alice to shut down with a reason, then approve the request and show the final request status."

Expected: request 有稳定 request_id；状态先是 pending，随后变为 approved 或 rejected，并写入对应 JSONL。

Step 7: 提交** 
git add go-agent/internal/protocols go-agent/internal/team go-agent/cmd/s10-team-protocols
git commit -m "feat(go): add team request protocols"
