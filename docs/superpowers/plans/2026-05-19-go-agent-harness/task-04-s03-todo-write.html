Task 4: 实现 s03 TodoWrite 和 nag reminder

Prerequisites: Task 3 (文件工具)

Goal: 添加会话内 Todo 管理工具，支持状态校验、渲染和 nag reminder hook。

给第一次接触 Agent 的读者

源教程 s03 解决的是“模型做多步任务容易忘步骤”的问题。TodoWrite 不是普通待办应用，而是给模型一个外部化的短期计划板：模型必须把计划写出来、标记当前正在做什么、完成后更新状态。

新手先懂这几个概念

  • 会话内状态：Todo 存在当前运行中的 manager 里，适合一次对话内的计划，不负责跨进程恢复。
  • 单一 in_progress：同一时间只允许一个任务进行中，迫使模型聚焦，不同时承诺多个动作。
  • Nag reminder：如果模型连续几轮不更新 todo，hook 会注入提醒。这是 harness 对模型行为的轻量约束。
  • 渲染状态：工具结果应该把 todo 列表以稳定文本返回，让模型和用户都能看到当前进度。

和源教程的对应关系

源 s03 把 todo 加进工具集合，并在 loop 旁边维护计数器。本任务用 internal/todo/manager.go 承载状态校验和渲染，用 agent hook 负责提醒注入，避免把规划逻辑写死在 loop 里。

完成后应该理解

你应该能区分“模型心里想的计划”和“harness 可检查的计划”。TodoWrite 的价值就在于把计划变成结构化状态，让测试可以验证，也让后续任务系统有清晰的对照。

Context

本任务在 Task 3 完成的 agent loop 和 tools 基础上，新增 todo 工具和 nag hook。Loop 的 BeforeCallHook 和 AfterToolHook 机制在 Task 2 中已定义。

Steps

Step 1: 写 Todo 测试**

覆盖：

  • 多个 in_progress 返回错误。
  • 非法状态返回错误。
  • 空 content 返回错误。
  • Render() 输出 [ ]、[>]、[x]。
Step 2: 实现 TodoManager**

字段：

type Item struct {
	Content    string `json:"content"`
	Status     string `json:"status"`
	ActiveForm string `json:"activeForm"`
}

Update(items []Item) (string, error) 校验状态并保存；最多 20 条。

Step 3: 注册 todo 工具和 nag hook**

实现：

  • todo.Register(reg, manager)。
  • agent.WithTodoNag(manager, threshold int) 返回 BeforeCallHook 和 AfterToolHook 所需状态。
  • 当连续 3 轮未调用 todo，向最新 user tool_result message 前插入 Update your todos.。
Step 4: 实现 s03 CLI**

system prompt 明确多步任务先调用 todo，同一时间只有一个 in_progress。

Run:

cd go-agent
go test ./internal/todo ./internal/agent
go run ./cmd/s03-todo-write "Refactor hello.go: add type hints equivalent comments, doc comments, and tests"

Expected: 测试 PASS；nag hook 单测可验证 reminder 注入。

Step 5: 实现后真实验证任务**

无网络验证要证明 todo 状态校验和 nag hook 都可测试：

cd go-agent
go test -count=1 ./internal/todo ./internal/agent ./internal/tools
go vet ./internal/todo ./internal/agent

真实模型验证要使用一个明确的多步任务，观察模型是否先写 todo、只保留一个 in_progress，并在完成后更新状态：

cd go-agent
go run ./cmd/s03-todo-write "Create tmp/todo-demo.txt, write three short lines, read it back, and keep your todo list updated."

Expected: 输出中出现 todo 列表从 [ ] 到 [>] 再到 [x] 的变化；连续不更新 todo 时能观察到 reminder 注入。
